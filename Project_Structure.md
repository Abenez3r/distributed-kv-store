# Project Structure - Distributed Key-Value Store

This document outlines the project structure for the Distributed Key-Value Store. It provides a breakdown of each file and directory, along with their purpose.


```plaintext
distributed-kv-store/
├── src/ 
│   ├── main/
│   │   ├── java/
│   │   │   └── com/
│   │   │       └── example/
│   │   │           └── kvstore/
│   │   │               ├── ConsistentHashing.java       # Implements consistent hashing for data distribution across nodes.
│   │   │               ├── LeaderNode.java              # Manages leader responsibilities, handles requests, and delegates operations.
│   │   │               ├── DataNode.java                # Represents individual nodes that store key-value pairs.
│   │   │               ├── RaftLeaderElection.java      # Handles leader election using a simplified Raft algorithm.
│   │   │               ├── Node.java                    # Defines the structure and behavior of a node in the system.
│   │   │               ├── HeartbeatService.java        # Sends heartbeat messages to ensure nodes are alive and responsive.
│   │   │               ├── SnapshotService.java         # Manages taking snapshots of the current key-value store for fault tolerance.
│   │   │               ├── CompressionService.java      # Provides methods to compress and decompress data stored in the system.
│   │   │               └── KeyValueStoreApp.java        # The main entry point of the application, initializes the system.
│   └── resources/
│       ├── application.properties                       # Application configuration properties for system settings.
│       ├── log4j2.xml                                   # Logging configuration using Log4j2.
│       └── node-config.json                             # Configuration for nodes in the distributed system.
├── test/
│   ├── java/
│   │    └── com/
│   │        └── example/
│   │            └── kvstore/
│   │               ├── ConsistencyTest.java             # Tests data consistency across nodes in the distributed system.
│   │               ├── DistributedSystemTest.java       # Integration tests for distributed system functionality.
│   │               ├── FaultToleranceTest.java          # Simulates node failures and verifies system fault tolerance.
│   │               ├── LatencyTest.java                 # Measures latency of key-value operations like put/get.
│   │               ├── LeaderNodeIntegrationTest.java   # Ensures the interaction between LeaderNode and other components works as expected.
│   │               ├── LeaderNodeTest.java              # Unit tests for leader node functionalities.
│   │               ├── PerformanceTest.java             # Tests the system under load to measure performance and throughput.
│   │               ├── ScalabilityTest.java             # Verifies the system’s ability to scale by adding/removing nodes.
│   │               └── SnapShotTest.java                # Tests the functionality of taking and restoring snapshots.
├── target/                                               # Contains compiled files and other build outputs (generated by Maven).
├── .gitignore                                            # Git ignore file to exclude unnecessary files from version control.
├── dependency-reduced-pom.xml                            # Dependency-reduced POM generated during the Maven build process.
├── Dockerfile                                            # Docker configuration file to containerize the application.
├── docker-compose.yml                                    # Docker Compose file for orchestrating multiple services (nodes) in Docker.
├── Makefile                                              # Makefile for automating build and testing tasks.
├── pom.xml                                               # Maven configuration file for project dependencies and build settings.
└── README.md                                             # Project documentation, providing an overview, installation instructions, and usage.
